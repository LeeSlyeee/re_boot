# Re:Boot 프로젝트 중간 점검 멘토링 피드백 분석 및 대응 계획

> **작성일**: 2026-02-19  
> **회의 예정일**: 2026-02-20  
> **목적**: 멘토링 피드백을 기반으로 현재 프로젝트 상태를 분석하고, 각 피드백 항목에 대한 구현 가능 범위와 의사결정 필요 사항을 정리

---

## 📌 1. 멘토링 피드백 원문 요약

### 1-1. 데이터 구조 및 UI/UX 개선

- **ERD 업데이트**: "모의면접" 관련 테이블/박스를 추가하여 면접 데이터 관리 구조 확립
- **Skill set 기능 강화**:
  - 사용자 편의를 위한 "정렬(Sorting)" 및 "검색" 기능 도입
  - 개별 스킬셋 클릭 시 관련 "학습 코스로 즉시 연결"되는 네비게이션 기능 추가

### 1-2. 학습 지원 기능 (학습자 측면)

- **STT 기반 2차 가공**: 실시간 강의/음성 인식(STT) 요약본을 "PDF 파일"로 내보내는 기능
- **노트 기능 고도화**: 요약본 내 "직접 필기" 및 내용을 추가할 수 있는 "Append(덧붙이기)" 기능
- **학습 지표**: 학생의 성실도를 확인할 수 있는 "출석률 지표" 시각화
- **타겟 확장**: 현재 설정된 '학생' 타겟층의 범위를 더 구체화하거나 확장할 필요가 있음

### 1-3. 관리자 페이지 (운영 및 데이터 관리)

- **리스크 관리**: 실시간 스트리밍/업로드의 기술적 리스크를 고려하여, "강의 녹음 후 가공된 결과물"을 별도로 업로드하는 프로세스 구축
- **통계 및 모니터링**:
  - 전체 학생의 "출석률 현황" 관리
  - 학생들이 푼 "퀴즈 데이터"를 효과적으로 시각화/표시

### 1-4. AI 모의면접 신뢰성 확보

- **평가 기준 수립**: 면접 점수 부여 시 논리적 근거가 되는 "내부 기준" 마련 필수
- **현실적 보완책**: AI 기본법 준수 및 실무적 신뢰성을 위해 "실제 전문가 자문"을 통한 기준 마련 필요
  - → 프로젝트의 전문성을 높이는 요소로, "추후 과제"로 명시하여 발전 가능성 제시 가능

## 📊 2. 현재 프로젝트 상태 분석

### 2-1. 기술 스택

| 구분              | 기술                                                 |
| ----------------- | ---------------------------------------------------- |
| Database          | MariaDB                                              |
| Backend           | Django REST Framework (DRF)                          |
| Frontend (학습자) | Vue.js 3 + Pinia + Vite 4 (Port: 5173)               |
| Frontend (강사)   | Vue.js 3 + Vite 5 (Port: 5174)                       |
| AI Engine         | STT(Whisper Chunking), LLM(GPT-4o/Claude 3.5 Sonnet) |

### 2-2. 이미 구현된 기능 (8주차까지 완료)

- ✅ 사용자 인증 (JWT/SimpleJWT) 및 Role 기반 분기 (STUDENT / INSTRUCTOR / MANAGER)
- ✅ STT Chunking 수집 및 AI 요약 파이프라인 (쉬는 시간 요약)
- ✅ 일일 퀴즈 생성 및 성취도 평가
- ✅ 하이브리드 학습 모드 (오프라인, 유튜브, 유니버설 캡처)
- ✅ 학습자 대시보드 + 성취도 그래프 (Chart.js)
- ✅ Backend-as-Truth 세션 복구 시스템
- ✅ 커리어 듀얼 아웃풋 포트폴리오 (취업/창업)
- ✅ AI 모의면접 (6종 페르소나, 채팅 UI)
- ✅ 강사 대시보드 (학생별 퀴즈 결과 조회)
- ✅ 분산형 수강 등록 시스템 (입장 코드)
- ✅ 실라버스 및 학습 목표 체크리스트

## ✅ 3. 피드백별 구현 가능 범위 분석

### 3-1. 즉시 구현 가능 (코드 레벨 착수 가능)

#### 📐 데이터 구조 및 UI/UX

| 피드백 항목                    | 현재 상태                                                                  | 구현 내용                                                                     |
| ------------------------------ | -------------------------------------------------------------------------- | ----------------------------------------------------------------------------- |
| **ERD에 모의면접 테이블 추가** | `MockInterview`, `InterviewExchange` 모델이 이미 `career/models.py`에 존재 | ERD 문서(`erd.md`)에 해당 테이블을 Mermaid 다이어그램으로 반영                |
| **Skill set 정렬/검색**        | `Syllabus`, `LearningObjective`, `StudentChecklist` 모델 존재              | 백엔드에 필터링/정렬 API (`?search=`, `?ordering=`) 추가 + 프론트엔드 검색 UI |
| **스킬셋 → 학습코스 연결**     | `LearningObjective` → `Syllabus` → `Lecture` FK 연결 존재                  | 프론트엔드에서 스킬 항목 클릭 → 해당 강의/세션으로 라우팅 네비게이션 추가     |

#### 📚 학습 지원 기능

| 피드백 항목                 | 현재 상태                                                           | 구현 내용                                                                        |
| --------------------------- | ------------------------------------------------------------------- | -------------------------------------------------------------------------------- |
| **STT 요약본 PDF 내보내기** | `SessionSummary.content_text`에 AI 요약 텍스트 저장                 | 백엔드 PDF 생성 API (Python `reportlab` / `weasyprint`) + "📄 PDF 다운로드" 버튼 |
| **노트 기능 (필기/Append)** | 요약본은 현재 읽기 전용                                             | `SessionSummary`에 `user_note` 필드 추가 또는 별도 `Note` 모델 + 메모 에디터 UI  |
| **출석률 지표 시각화**      | `LearningSession`에 `session_date`, `is_completed` 출석 데이터 존재 | 백엔드 출석률 집계 API + 대시보드에 Chart.js 출석률 차트                         |

#### 🖥️ 관리자 페이지

| 피드백 항목                        | 현재 상태                                            | 구현 내용                                                     |
| ---------------------------------- | ---------------------------------------------------- | ------------------------------------------------------------- |
| **강의 녹음 → 가공 결과물 업로드** | 현재 실시간 STT만 지원                               | 오디오 파일 업로드 API + 서버사이드 Whisper 처리 파이프라인   |
| **전체 학생 출석률 현황**          | `Professor_dashboard`에 `LectureDetailView.vue` 존재 | 강사 대시보드에 전체 학생 출석률 테이블/차트 추가             |
| **퀴즈 데이터 시각화**             | `QuizAttempt`, `AttemptDetail` 모델 존재             | 학생별 퀴즈 점수 분포 차트 (Bar/Radar) + 문항별 정답률 시각화 |

#### 🎤 AI 모의면접

| 피드백 항목                | 현재 상태                                              | 구현 내용                                                         |

### 3-2. ⚠️ 의사결정 필요 항목 (설계 승인 후 구현)

#### 항목 A: 타겟 확장 (학생 → ?)

**❓ 핵심 질문**: "누구를 위한 서비스인가?"

현재 User 모델의 Role 구조:

```python
class Role(models.TextChoices):
    STUDENT = 'STUDENT'       # 부트캠프 수강생
    INSTRUCTOR = 'INSTRUCTOR' # 강사
    MANAGER = 'MANAGER'       # 매니저
```

**선택지와 기술적 영향:**

| 선택지                    | 의미                                            | DB/API/UI 변경 범위                                                        |
| ------------------------- | ----------------------------------------------- | -------------------------------------------------------------------------- |
| **A. 현행 유지 (구체화)** | "부트캠프 수강생" 타겟 유지, 프로필 필드 세분화 | `User` 모델에 프로필 필드 추가. **변경 최소**                              |
| **B. 일반 학습자로 확장** | 독학/온라인 강의 수강자 포함                    | `Lecture` 모델 의미 변경. 강사 없는 자율학습 흐름 추가. **중간 규모 변경** |
| **C. 기업 교육으로 확장** | 사내 직원 역량 강화 교육                        | `COMPANY_ADMIN` Role 추가. Organization 모델 신설. **대규모 구조 변경**    |
| **D. 취준생 + 재직자**    | 이직 준비 재직자 포함                           | `goal_type` 필드 적극 활용. 커리어 트랙별 분기 강화. **소~중간 규모 변경** |

> **➡️ 결정 필요**: 방향에 따라 DB 모델 구조가 근본적으로 달라지므로, 기획 의사결정 후 구현 착수

#### 항목 B: AI 면접 평가 기준 — 전문가 자문 및 AI 기본법 대응

**❓ 핵심 질문**: "AI가 만든 초안 Rubric으로 우선 진행해도 되는가? AI 기본법 준수는 어떻게 할 것인가?"

**할 수 있는 것 vs 할 수 없는 것:**

| 구분      | 내용                                                                                                   |
| --------- | ------------------------------------------------------------------------------------------------------ |
| ✅ 가능   | 다차원 평가 Rubric 모델 설계 (기술깊이 / 논리성 / 소통력 등)                                           |
| ✅ 가능   | AI 프롬프트에 구체적 채점 기준표 주입 (STAR 기법, Google Hiring Attribute 등 참고)                     |
| ✅ 가능   | 결과 화면에 Disclaimer 표시: _"본 평가는 AI 자동 생성 참고 자료이며, 전문가 검증을 거치지 않았습니다"_ |
| ✅ 가능   | 발표/문서에 "추후 과제: 전문가 자문 기반 고도화" 섹션 명시                                             |
| ✅ 가능   | AI 활용 **사전 고지** UI 구현 ("이 서비스는 AI를 활용합니다")                                          |
| ✅ 가능   | AI **생성물 표시** 라벨 부착 (면접 질문·평가 결과에 "AI 생성" 태그)                                    |
| ✅ 가능   | 면접 결과 **비결정성 고지** ("이 결과는 실제 채용 결과와 무관합니다")                                  |
| ❌ 불가능 | 실제 면접관/HR 전문가 섭외 및 자문                                                                     |
| ⏳ 추후   | 법률 전문가 자문을 통한 공식 컴플라이언스 판단                                                         |
| ⏳ 추후   | 과기정통부 "고영향 AI 해당 여부 확인 요청" (신청 시 30일 내 회신)                                      |

> **➡️ 현실적 제안**: 업계 통용 프레임워크(STAR 기법 등) 기반 초안 Rubric 생성 + Disclaimer 부착으로 진행.  
> 멘토도 "추후 과제로 명시하라"고 했으므로, 이 방식이 적절

#### 항목 C: AI 기본법 적용 분석 (2026.01.22 시행)

**❓ 핵심 질문**: "Re:Boot의 AI 모의면접이 AI 기본법상 어떤 규제 범주에 해당하는가?"

**고영향 AI 해당 여부 판단 (3가지 조건 모두 충족 시 해당):**

| 조건                  | Re:Boot 적용                                                           | 판단                           |
| --------------------- | ---------------------------------------------------------------------- | ------------------------------ |
| ① 10개 법정 영역 해당 | "채용"·"교육" 영역에 인접하나, 직접적인 채용 프로세스가 아닌 연습 도구 | ⚠️ 인접하나 직접 해당은 불분명 |
| ② 중대한 영향         | 실제 채용 합격 여부를 결정하지 않음. 학습 보조 목적                    | 🟢 제한적                      |
| ③ 100% AI 판단        | 실제 채용 결정을 내리지 않고, 참고용 피드백만 제공                     | 🟢 해당 안됨                   |

> **➡️ 결론**: 세 조건을 동시에 충족하지 않으므로 **고영향 AI에 해당할 가능성은 낮음**

**생성형 AI 해당 여부:**

| 항목                     | 판단        | 근거                                       |
| ------------------------ | ----------- | ------------------------------------------ |
| GPT-4o / Claude 3.5 활용 | ✅ **해당** | AI가 텍스트(면접 질문, 평가 피드백)를 생성 |

> **➡️ 결론**: 생성형 AI 관련 **투명성 의무**(사전 고지, 생성물 표시)는 **적용 가능성 있음**

**즉시 구현 가능한 컴플라이언스 대응 (UI 문구 수준, 구현 난이도 낮음):**

| 항목                  | 구현 내용                                                               | 위치                         |
| --------------------- | ----------------------------------------------------------------------- | ---------------------------- |
| **AI 활용 사전 고지** | 모의면접 시작 전 "이 서비스는 AI를 활용합니다" 안내 표시                | `InterviewChatView.vue` 상단 |
| **AI 생성물 표시**    | 면접 질문·평가 피드백에 "AI 생성 콘텐츠" 라벨 부착                      | 채팅 버블 / 결과 리포트      |
| **Disclaimer 표시**   | "본 평가는 AI 자동 생성 참고 자료이며, 전문가 검증을 거치지 않았습니다" | 결과 리포트 하단             |
| **비결정성 고지**     | "이 결과는 실제 채용 결과와 무관합니다"                                 | 결과 리포트 하단             |

**계도 기간 참고:** AI 기본법은 시행 후 **최소 1년(~2027.01)** 계도 기간 운영 (과태료 유예). 위 기본 대응만 갖춰도 현 시점에서 충분하며, 추후 전문가 자문을 통해 고도화 가능
