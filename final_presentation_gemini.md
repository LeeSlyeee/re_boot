# 🚀 Re:Boot 프로젝트 최종 발표 프리젠테이션 대본 및 구성안

## 1. Re:Boot 소개

**[슬라이드 1: 타이틀 및 도입]**

안녕하세요! 맞춤형 커리어 빌드업과 지능형 학습 관리를 지원하는 플랫폼, **Re:Boot** 프로젝트의 발표를 맡은 [발표자 이름]입니다.

**[슬라이드 2: Pain point (우리가 발견한 문제점)]**

온라인과 오프라인을 넘나드는 현재의 교육 환경에서 우리는 다음과 같은 치명적인 **세 가지 문제점(Pain Point)**을 발견했습니다.

1. **학생의 몰입도 저하와 정보 누락**: 긴 강의를 듣다 보면 중요한 흐름을 놓치기 쉽고, 결석이라도 한 날엔 방대한 학습량을 스스로 따라잡기 매우 막막합니다.
2. **개인화된 피드백의 한계**: 한 명의 강사가 수십 명 학생의 수준을 일일이 파악해 맞춤형 지도를 제공하는 것은 물리적으로 불가능합니다.
3. **학습 성과의 증명 부재**: 열심히 공부를 해도, "내가 실제로 무엇을 할 수 있는지"를 취업 시장에 매력적인 포트폴리오로 증명하기 어렵습니다.

**[슬라이드 3: Solution (우리의 해결책)]**

Re:Boot는 이 문제를 기술적으로 해결합니다.

1. **실시간 AI 자막 & 퀴즈 자동 생성**: 수업 중 강사의 음성을 실시간으로 텍스트로 변환(STT)해 요약 노트를 만들고, 수업 직후 배운 내용을 점검하는 일일 퀴즈를 AI가 즉시 생성합니다.
2. **데이터 기반의 초개인화 학습**: 개개인의 퀴즈 정답률과 이해도를 바탕으로 '갭 맵(Gap Map)'을 구성하여, 본인이 부족한 역량이 무엇인지 시각적으로 파악하고 'AI 복구 플랜'을 통해 지름길을 안내받습니다.
3. **스킬블록 기반 포트폴리오 시스템**: 강의 목표 달성도를 조각조각 모아 '스킬 블록'으로 만들고, 이를 최종적으로 면접 대비를 위한 포트폴리오로 체계화해 줍니다.

**[슬라이드 4: Core Value (핵심 가치)]**

즉, **Re:Boot**의 핵심 가치는 **"Seamless(끊김 없는 몰입)"**과 **"Tailored(극강의 초개인화)"**입니다. 수동적인 학습자를 능동적인 포트폴리오 소유자로 리부트시키는 것, 그것이 저희가 제안하는 가치입니다.

## 2. 4주간의 개발 과정

**[슬라이드 5: 타임라인 및 마일스톤]**

저희 팀은 짧은 4주라는 시간 동안 밀도 있게 프로젝트를 완성했습니다.

- **1주차 (기획 & 설계)**: 사용자 여정(User Journey) 매핑, 피그마(Figma) UI/UX 디자인, 요구사항 명세 및 데이터베이스(ERD) 설계. 핵심 도메인을 '학습-라이브-분석'으로 정의했습니다.
- **2주차 (핵심 백엔드 & AI 파이프라인 구축)**: Django REST Framework를 기반으로 코어 API를 세팅하고, STT 엔진(라즈베리파이 연동) 및 OpenAI GPT-4o를 이용한 텍스트 분석/퀴즈 생성 파이프라인을 구축했습니다.
- **3주차 (프론트엔드 연동 & 대시보드 시각화)**: Vue.js와 TailwindCSS를 활용해 학생용 웹앱과 교수자용 대시보드를 분리 개발했습니다. Chart.js를 이용해 통계 데이터를 직관적으로 시각화하는 데 집중했습니다.
- **4주차 (통합 테스트 & CI/CD 배포 최적화)**: OCI(Oracle Cloud Infrastructure)에 Nginx 셋업 및 502/CORS 이슈 대응, 그리고 실제 네트워크 지연 상황을 가정한 예외 처리(Fallback UX) 강화 및 최종 디버깅을 진행했습니다.

## 3. Service Flow (서비스 흐름)

**[슬라이드 6: Student Flow (학생의 여정)]**

학생의 시점에서 서비스는 이렇게 흘러갑니다.

1. **온보딩 & 진단**: 최초 접속 시 희망 직무 목표(예: AI 엔지니어)를 설정하고, 진단 고사를 통해 본인의 현재 레벨을 부여받습니다.
2. **라이브 세션 참여**: 수업에 입장하면 실시간으로 생성되는 자막을 보며 학습합니다.
3. **일일 복습 및 체크리스트**: 수업이 끝나면 AI가 요약한 노트와 생성된 퀴즈를 풀며 당일의 '학습 목표 체크리스트'를 달성합니다.
4. **포트폴리오 빌드**: 달성한 체크리스트가 누적되어 갭맵을 채우고, 최종적으로 수료증과 자기소개서용 스킬블록으로 변환됩니다.

**[슬라이드 7: Instructor Flow (교수자의 여정)]**

반면 교수자의 대시보드에서는 다음과 같은 흐름이 제공됩니다.

1. **클래스 룸 개설 및 진단**: 수업을 개설하고 학습 목표(Syllabus)를 등록합니다. 현재 클래스 학생들의 수준 분포를 도넛 차트로 한눈에 파악합니다.
2. **모니터링**: 퀴즈 정답률, 출석 현황 등 학생들의 성취도를 실시간으로 추적합니다.
3. **분석 및 개입**: AI가 분석해준 "공통 취약 역량 TOP 5"를 보고 다음 강의 방향을 수정하거나, 진도가 밀리는 특정 학생(위험군)을 타겟하여 피드백 메시지나 보충 자료를 클릭 한 번에 전송합니다.

## 4. 라이브 데모 시나리오 (시연 영상)

**[슬라이드 8: 시연 시나리오 소개]**

지금부터 말씀드린 기능이 실제 어떻게 작동하는지 영상으로 보여드리겠습니다.

- **Scene 1. (교수자 뷰) 진단 및 세션 시작**: 교수가 대시보드에서 수업을 열고, 학생들의 사전 레벨 진단(초급/중급/고급) 분포를 확인합니다.
- **Scene 2. (학생/교수 동시 뷰) 라이브 STT**: 교수가 마이크로 말을 시작하면(STT 엔진 가동), 학생의 화면에 실시간으로 자막이 렌더링 됩니다.
- **Scene 3. (학생 뷰) AI 퀴즈 및 복구 플랜**: 강의 종료 후, 학생이 요약 노트를 보고 AI가 자동 생성한 퀴즈를 풉니다. 퀴즈를 틀리면 진도율이 떨어지고, [🚑 복구 플랜] 버튼을 누르면 "자신이 놓친 핵심 개념 3분 요약 가이드"를 AI가 즉석에서 생성해줍니다.
- **Scene 4. (학생 뷰) 인스타 공유용 수료증**: 모든 학습 목표를 체크한 학생이 '수료증' 메뉴로 들어가 인쇄 및 다운로드 버튼을 눌러 소셜 공유를 준비하는 장면입니다.

## 5. 시스템 아키텍처 및 특장점

**[슬라이드 9: System Architecture (단순화된 블록도)]**

저희 시스템의 뼈대는 확장성과 안정성에 초점을 맞췄습니다.

- **Frontend**: Vue 3를 반응형(Reactivity) 중심으로 활용하여, 교수용/학생용 앱 모두 끊김없는 SPA(Single Page Application)로 구현했습니다.
- **Backend (API + Data Layer)**: Python Django 기반의 RESTful API를 제공하며, MariaDB를 메인 스토어로 사용합니다.
- **AI & Hardware Edge**: 라즈베리파이를 활용한 로컬 Edge 환경에서 교수자의 음성을 일차적으로 수집 및 전처리(KWS)하고, 클라우드의 핵심 AI(OpenAI GPT-4o)가 요약과 포트폴리오 생성을 담당하는 '하이브리드 AI 아키텍처'를 띄고 있습니다.

**[슬라이드 10: 핵심 기술 특장점]**

1. **Dynamic Prompting**: 단순 API 호출을 넘어, 학생의 '현재 레벨(Level 1~3)', '과거 오답 기록'을 템플릿에 동적으로 주입하여 수준별로 다른 텍스트를 응답하는 고도화 프롬프트 엔지니어링을 적용했습니다.
2. **내결함성(Fault-tolerance) 대비**: 만약 외부 AI API 통신이 지연되거나 실패하더라도 시스템이 다운되지 않도록, 백엔드 로직에 Fallback(임시 정적 데이터) 응답 구조를 모두 구현해두었습니다. 기능이 멈추는 일은 없습니다.

## 6. 기대 효과 및 비즈니스 임팩트

**[슬라이드 11: 비즈니스 가치]**

Re:Boot 솔루션이 에듀테크 시장에 도입된다면 다음과 같은 파급력을 예상합니다.

- **B2C (수강생)**: 강의 이탈률(Drop-out rate)을 획기적으로 줄이고, 학습의 결과를 눈에 보이는 '가시적 형태(수료증, 이력서 스킬셋)'로 환산해주어 플랫폼 충성도와 완강률을 극대화합니다.
- **B2B (학원, 교육 기관)**: 강사가 반복적인 서무(질문 답변, 퀴즈 출제, 개별 독려)에 쏟는 시간을 AI가 70% 이상 단축시켜 줍니다. 강사는 '교육 컨텐츠의 퀄리티' 자체에만 집중할 수 있게 되어 기관 전체의 프리미엄 교육 역량이 상승합니다.

## 7. 한계점 및 향후 계획

**[슬라이드 12: Next Step]**

4주라는 시간 특성상 분명히 아쉬운 한계점도 존재했으며, 이를 어떻게 발전시킬지 고민했습니다.

- **실시간 음성 트래픽 최적화**: 현재 라즈베리파이망과 서버 간의 STT 소켓 통신에서 네트워크 불안정 시 일부 딜레이가 발생할 때가 있습니다. WebRTC 기반의 전용 스트리밍 오디오 채널을 도입하여 완전한 실시간성을 보장할 계획입니다.
- **채점 로직의 고도화**: 현재 객관식/단답형 위주로 구성되어 있는 퀴즈 시스템을, RAG(검색 증강 생성) 기반으로 진화시켜 "학생이 서술형으로 답변한 텍스트" 문맥의 의도까지 파악하여 부분 점수를 부여하는 서술형 평가 모델을 추가할 계획입니다.
- **모바일 네이티브 전환**: PWA 수준을 넘어, 기기 마이크와 카메라에 완벽히 밀착되는 React Native 형태의 앱으로 컨버팅해 접근성을 높일 것입니다.
