## 시나리오 1️⃣ — 학생: 진단 테스트 하기

### 준비

- `testuser`로 학생 프론트엔드(http://localhost:5173)에 **로그인**

### 수행 절차

1. 좌측 메뉴 또는 URL로 **`/placement`** 페이지 이동
2. 진단 문항 20개가 표시되는지 확인

### 확인 포인트

- [ ] 문항이 화면에 표시됨 (20개)
- [ ] 각 문항에 객관식 보기가 있음
- [ ] 문항마다 하나를 선택할 수 있음

---

## 시나리오 2️⃣ — 학생: 레벨 판정 확인

### 수행 절차

1. 문항에 답을 선택 (자유롭게)
2. **제출** 버튼 클릭

### 확인 포인트

- [ ] 결과 화면이 나타남
- [ ] **레벨**이 표시됨 (Level 1 / Level 2 / Level 3)
- [ ] **점수**와 **정답률**이 표시됨
- [ ] **카테고리별 점수**(개념/실습/패턴)가 보임

### 레벨 판정 기준 참고

| 정답률 |        판정 레벨        |
| :----: | :---------------------: |
| 0~39%  | Level 1 - 쉽게 이해하기 |
| 40~69% |   Level 2 - 핵심 정리   |
|  70%+  |   Level 3 - 심화 완성   |

## 시나리오 3️⃣ — 학생: 갭 맵 확인

### 수행 절차

1. 진단 완료 후 **`/gapmap`** 페이지 이동

### 확인 포인트

- [ ] 카테고리별 스킬 목록이 표시됨
- [ ] 각 스킬에 상태가 표시됨 (보유 / 학습중 / 부족)
- [ ] 전체 완성도(%) 통계가 보임
- [ ] Level 1로 판정받은 경우: "부족(GAP)" 스킬이 많음
- [ ] Level 3으로 판정받은 경우: "보유(OWNED)" 스킬이 많음

## 시나리오 4️⃣ — 학생: 재접속 시 결과 유지

### 수행 절차

1. 브라우저를 **새로고침**하거나 다시 `/placement` 접속

### 확인 포인트

- [ ] 문항이 다시 나오지 않고, **이전 결과 화면**이 바로 표시됨
- [ ] 레벨, 점수, 정답률이 이전 제출 결과와 동일함

## 시나리오 5️⃣ — 교수자: 학생 레벨 분포 확인

### 준비

- `professor`로 교수자 대시보드(http://localhost:5174)에 **로그인**

### 수행 절차

1. **Vue.js 기초** 강의 선택
2. 진단 현황 / 학생 분석 영역 확인

### 확인 포인트

- [ ] 수강생 중 진단 완료한 학생 수 표시
- [ ] Level 1 / Level 2 / Level 3 인원 분포 표시
- [ ] 취약 역량 TOP 5 목록 표시

---

## 시나리오 6️⃣ — 교수자: 교안 AI 변형 + 레벨 연동

### 수행 절차

1. 교수자 대시보드에서 **교안 관리** 섹션 확인
2. 교안을 업로드 (MD, TXT, PDF, DOCX 등)
3. **🤖 AI 변형 생성** 버튼 클릭
4. 생성된 3개 레벨(쉽게 이해하기 / 핵심 정리 / 심화 완성) 카드 확인
5. 각 레벨 카드 **클릭** → 미리보기 모달 확인
6. 내용 확인 후 **✅ 승인** 버튼 클릭

### 확인 포인트

- [ ] 3개 레벨(🌱📚🚀)이 카드 형태로 표시됨
- [ ] 클릭 시 전체 내용이 마크다운으로 렌더링됨
- [ ] Level 1은 쉬운 용어 + 비유 + 이모지 위주
- [ ] Level 2는 원본 구조 유지 + 연습문제 포함
- [ ] Level 3은 배경 지식 + 주의사항 + 실전 문제 포함
- [ ] 승인 후 상태가 "📝 초안" → "✅ 승인됨"으로 변경됨
