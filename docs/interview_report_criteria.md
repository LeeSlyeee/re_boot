## 4. 차원별 분석 (Rubric — 4개 차원)

### 4.1 평가 차원 정의

| #   | 차원 (key)          | 한글명        | 아이콘 | 평가 관점                                           |
| --- | ------------------- | ------------- | ------ | --------------------------------------------------- |
| 1   | `technical_depth`   | 기술적 깊이   | 🔬     | 개념의 정확성, 기술 선택의 이유, 대안 기술과의 비교 |
| 2   | `logical_coherence` | 논리적 일관성 | 🧠     | 답변 구조, STAR 기법 준수, 인과관계의 명확성        |
| 3   | `communication`     | 소통 능력     | 💬     | 설명의 명확성, 적절한 예시 활용, 질문 의도 파악     |
| 4   | `problem_solving`   | 문제 해결력   | 🔧     | 실무 적용 능력, 예외/엣지 케이스 고려, 창의적 접근  |

> **STAR 기법**: Situation(상황) → Task(과제) → Action(행동) → Result(결과)  
> 구조화된 답변 기법으로, 실제 채용 면접에서도 널리 사용됩니다.

### 4.2 차원별 점수

| 항목                  | 값                                                         |
| --------------------- | ---------------------------------------------------------- |
| **각 차원 점수 범위** | 0 ~ 100 (문항별)                                           |
| **산출 주체**         | AI가 문항별로 각 차원을 채점                               |
| **리포트 표시값**     | 모든 문항의 해당 차원 점수를 **산술 평균** (서버에서 계산) |
| **코멘트**            | 문항별로 AI가 1줄 근거를 함께 제공 (채팅 중 피드백에 표시) |

### 4.3 점수 해석 가이드

| 점수대 | 해석                                         |
| ------ | -------------------------------------------- |
| 90~100 | 해당 역량이 매우 뛰어남. 실무 투입 가능 수준 |
| 70~89  | 기본기가 탄탄하며, 약간의 보완으로 실무 가능 |
| 50~69  | 기초 개념은 있으나, 깊이있는 학습이 필요     |
| 30~49  | 개념 이해가 부족하며, 체계적인 학습 필요     |
| 0~29   | 해당 영역에 대한 사전 지식이 거의 없음       |

## 7. 학습 ↔ 면접 연결 (Skill Connection)

### 7.1 진행률 수치

| 항목                | 산출 방식                                                        |
| ------------------- | ---------------------------------------------------------------- |
| **스킬 진행률 (%)** | `(획득한 블록 수 / 전체 블록 수) × 100`, 소수점 반올림           |
| **획득한 블록**     | 수강 강의의 학습 목표 중 `StudentChecklist.is_checked=True`인 수 |
| **전체 블록**       | 수강 강의(Lecture)에 등록된 전체 `LearningObjective` 수          |
| **미획득 블록**     | `전체 - 획득` (프론트엔드에서 계산)                              |

### 7.2 카테고리별 현황

| 항목                 | 설명                                                      |
| -------------------- | --------------------------------------------------------- |
| **카테고리 기준**    | 강의(Lecture) 단위로 그룹핑                               |
| **막대 그래프 색상** | 80%↑ 파랑, 50~79% 노랑, 50%↓ 빨강 (점수 색상 기준과 동일) |
| **표시 형식**        | `Vue.js 기초 ██████░░ 17/24`                              |

### 7.3 추천 스킬 블록

| 항목               | 값                                                    |
| ------------------ | ----------------------------------------------------- |
| **선정 기준**      | 미획득 학습 목표 중 **낮은 주차 순** (선행 학습 우선) |
| **최대 표시 개수** | 5개                                                   |
| **표시 내용**      | 학습 목표명 + 소속 강의명                             |
| **레벨 이모지**    | 🌱 (기본값, 현재 모든 추천은 level=1)                 |

### 7.4 동기부여 메시지 분기

| 조건              | 메시지                                    | CTA 버튼                              |
| ----------------- | ----------------------------------------- | ------------------------------------- |
| **진행률 ≥ 100%** | "🎉 모든 스킬 블록을 획득했습니다!"       | 📄 포트폴리오 생성하기 → `/portfolio` |
| **진행률 < 100%** | "💡 학습이 곧 면접 경쟁력!" + 현재 % 표시 | 📚 학습하러 가기 → `/dashboard`       |

### 7.5 데이터 실시간성

| **스킬 진행률** (획득/전체/%)     | **매번 실시간 DB 쿼리** (학습 진도 변동 즉시 반영) |

## 8. 채팅 중 피드백 (문항별)

면접 진행 중 각 답변에 대해 표시되는 피드백:

| 항목               | 내용                              |
| ------------------ | --------------------------------- |
| **종합 점수 뱃지** | 문항의 `overall_score` (0~100)    |
| **종합 피드백**    | AI가 작성한 2~3줄의 피드백 텍스트 |
| **Rubric 바 차트** | 4개 차원별 점수 + 1줄 코멘트      |

### 채팅 중 종합 점수 산출

- `msg.score`가 존재하면 그대로 사용
- 없으면 4개 차원 점수의 **산술 평균** (프론트엔드에서 계산)

## 10. 전체 데이터 흐름 요약

```
[면접 시작]
  └→ 포트폴리오 + 스킬 목록 → AI가 첫 질문 생성

[답변 제출 (문항별)]
  └→ AI가 4차원 Rubric 채점 (0~100 × 4) + overall_score + 피드백 + 다음 질문

[면접 종료]
  └→ 서버: 모든 문항 점수 집계 (차원별 평균, 종합 평균, Best/Worst)
  └→ AI: 종합 분석 생성 (등급, 강점, 약점, 개선 팁, 추천 학습)  ← 1회 캐싱
  └→ DB: 스킬 진행률 실시간 계산 (StudentChecklist 기반)       ← 매번 실시간
  └→ 최종 리포트 JSON 반환
```
