# Re:Boot 프로젝트 전체 개요서

> **서비스명**: Re:Boot (리부트) — AI 실시간 경로 재설계 및 학습 경험 자산화 커리어 빌드업 플랫폼
> **작성일**: 2026-03-13
> **현재 상태**: MVP v1.0 (OCI 클라우드 배포 완료, 8단계 통합 테스트 통과)

---

## 1. Re:Boot이란 무엇인가

### 한 줄 정의

Re:Boot은 **부트캠프 및 직업 교육 현장**에서 학습자의 실시간 강의 음성을 AI가 자동으로 필기·요약하고, 학습 진도를 동적으로 재설계하며, 모든 학습 경험을 커리어 자산으로 전환해주는 **AI 기반 학습 지원 SaaS** 플랫폼이다.

### 핵심 철학

기존의 모든 LMS(Learning Management System)는 **"무엇을 가르칠 것인가"** 또는 **"어떻게 관리할 것인가"**를 고민한다. Re:Boot은 질문 자체가 다르다:

**"어떻게 끝까지 완주시키고, 그 경험을 커리어 자산으로 전환할 것인가?"**

이 질문에 데이터와 AI로 답하는 것이 Re:Boot의 존재 이유이다.

### 기획 배경 — 왜 지금인가?

1. **보이지 않는 격차**: 커리큘럼이 요구하는 수준과 학습자의 현재 수준 사이의 괴리를 메워줄 세밀한 가이드가 부재하다.
2. **단절된 경험과 중도 이탈**: 잠시 학습을 쉬면 진도를 따라잡을 방법이 없어 영구적인 포기로 이어지며, 이는 국가 교육 예산의 낭비로 직결된다.
3. **경험의 증발**: 과정을 수료하지 못하면 그동안 쌓은 지식까지 모두 "실패"로 규정되어 버려진다.
4. **IP 보호와 상생의 필요성**: 오프라인 강의의 디지털화 과정에서 강사의 지적재산권을 보호하면서도 학습 효율을 높일 수 있는 상생 모델이 절실하다.

## 2. 해결하고자 하는 5대 문제 (Pain Points)

| # | 문제 | 현재 상황 | Re:Boot의 해결 방식 |
|:-:|:-----|:---------|:-------------------|
| 1 | **수준 파악 불가** | 이상적 커리큘럼 vs 학습자 현재 위치 간 격차 확인 불가 | 실시간 퀴즈 + 이해도 펄스로 진단 |
| 2 | **단계 부재** | 수업 후 다음 목표나 중간 단계를 제시해주는 시스템 없음 | 자동 체크리스트 + 스킬 블록 |
| 3 | **관리의 한계** | 온라인 강의 특성상 의지 부족 학생 밀착 케어 불가능 | 골든타임 알림 + 이탈 예측 AI |
| 4 | **재시작 가이드 부재** | 중도 포기 후 복귀 시 계획 재수립이 어려움 | 동적 리라우팅 (내비게이션 방식) |
| 5 | **매몰 비용** | 중도 하차 시 학습 기록이 전부 증발 | 스킬 블록 자산화 + 듀얼 포트폴리오 |

## 3. 핵심 기능 7가지 상세

### 기능 1: 몰입형 AI 학습 비서 (Focus-First Environment)

- 앱 실행만으로 강의 음성을 인식(STT)하여 "나만의 디지털 교과서"를 자동 생성한다.
- **Whisper Chunking**: 1분 단위로 음성을 분할하고, Sentence Buffering으로 문장 완성을 검증한다.
- **쉬는 시간 자동 요약**: 강의 중간 쉬는 시간마다 GPT-4o가 핵심 요약을 생성한다.
- **자동 체크리스트**: 당일 학습 내용 기반으로 "오늘 확인해야 할 개념" 체크리스트를 자동 생성한다.
- **IP 보호 전략**: 강사의 원본 음성은 저장하지 않고, AI가 재구성한 2차 저작물(요약/체크리스트)만 제공하여 지적재산권을 보호한다.

### 기능 2: Context-Aware AI 튜터

- 단편적 질문-답변이 아니라, 현재 수업 맥락(Context)을 기억하고 답변하는 AI 챗봇이다.
- RAG(Retrieval-Augmented Generation) 기반으로 당일 STT 로그를 벡터 DB에 저장하고, 학생 질문에 대해 수업 내용과 맥락을 반영한 답변을 제공한다.
- 예시: "아까 강사님이 말한 예시 다시 설명해줘" → 해당 STT 구간을 찾아 맥락 답변

### 기능 3: 일일 퀴즈 자동 출제 시스템

- 당일 STT 요약 내용을 기반으로 AI가 퀴즈를 자동 생성한다.
- 퀴즈 정답률 + 학습 속도 + 이탈 기간 데이터를 분석하여 학습자의 이해도를 정량적으로 측정한다.
- 기존 LMS(Moodle, Canvas)의 퀴즈는 교수자가 수동으로 출제하지만, Re:Boot은 "당일 학습 내용"을 기반으로 AI가 자동 출제한다는 점이 근본적 차이이다.

### 기능 4: 동적 리라우팅 엔진 (Dynamic Re-routing)

- 자동차 내비게이션처럼 학습자의 현재 위치와 공백기를 분석한다.
- 경로를 이탈했을 때 "경로를 재탐색합니다"라는 메시지와 함께, 현재 시점에서 목표(취업/창업)에 도달할 수 있는 "압축 패스트 트랙"으로 커리큘럼을 실시간 수정 제안한다.
- **트리거 조건**: 7일 연속 미접속, 퀴즈 3연속 50% 미만, 세션 2주 이상 미완료
- **대응**: 커리큘럼 순서 재배열 + 압축 복구 플랜 자동 생성
- 기존 LMS인 Moodle의 "적응형 학습 경로"가 가장 유사하나, 이는 교수자가 사전 설계한 분기 조건 기반이다. Re:Boot의 리라우팅은 학습자의 실시간 데이터를 AI가 분석하여 자동으로 커리큘럼을 수정한다는 점에서 근본적으로 다르다.

### 기능 5: 스킬 블록 자산화 (Skill-Block Asset)

- 완강 여부와 관계없이 학습한 구간까지의 성취를 시각화된 "디지털 자산"으로 변환한다.
- 기존 LMS에서 중도 포기는 "실패"로 처리되고 모든 기록이 소실된다. Re:Boot은 부분적 성취를 데이터로 보존하여, "끝까지 못 갔지만, 당신은 Python까지 배운 사람"이라는 메시지를 전달한다.
- 부트캠프 중도 탈락률이 15-30%인 현실에서, 이 학습 데이터의 자산화는 국가 교육 예산 효율화와도 직결된다.

### 기능 6: 듀얼 아웃풋 포트폴리오 (Dual-Output Portfolio)

- **취업 모드 (Career Report)**: 학습 로그를 기반으로 역량 키워드와 결과물을 종합한 "데이터 증빙형 포트폴리오" AI 자동 생성
- **창업 모드 (MVP Spec)**: 습득한 기술을 사업 아이템에 매칭하여 기술 명세서 및 "MVP 기획 초안" 자동 생성
- Coursera의 Professional Certificate가 가장 유사하지만, 이는 플랫폼이 사전 설계한 고정 자격증이다. Re:Boot은 학습자 개인의 실제 학습 데이터를 기반으로 취업/창업 목적별 맞춤 포트폴리오를 AI가 조합한다.

### 기능 7: AI 모의면접 시스템

- 6종 면접관 페르소나(시니어 개발자, 친절한 사수, 인사 담당자, 스타트업 대표, 빅테크 면접관, 압박 면접관) 제공
- 학습자의 포트폴리오 + 스킬 목록을 기반으로 AI가 맞춤형 면접 질문을 생성한다.
- 4차원 평가 기준(기술적 깊이/논리적 일관성/소통 능력/문제 해결력) 기반 채점 + 종합 리포트 제공

## 4. 서비스 사용 시나리오

### 학습자 관점

1. **진단 및 설정**: 부트캠프 입장 코드(6자리)로 등록, 레벨 테스트로 초급/중급/고급 분류
2. **몰입 학습**: 앱을 켜두면 AI가 실시간으로 요약 노트를 작성하고 체크리스트를 생성
3. **위기 관리**: 진도가 밀리거나 이해도가 낮아지면 AI가 즉시 "압축 복구 플랜"을 가동하여 커리큘럼을 수정
4. **자산화**: 학습 종료 시(수료 혹은 중단), 획득한 스킬 블록을 기반으로 "커리어 리포트"를 발급받아 활용

### 강사 관점

1. **강의 대시보드**: 학생별 성취도 차트, 이탈 위험군 모니터링
2. **실라버스 인라인 편집**: 학습 목표와 체크리스트 실시간 수정
3. **피드백 수신**: 학생들이 공통적으로 어려워하는 구간이나 학습 부진 지점을 데이터로 추출하여 "강의 개선 인사이트 리포트"를 역으로 제공
4. **녹음 업로드**: 강의 녹음 파일을 업로드하면 서버에서 Whisper 처리 → 요약 → RAG 인덱싱까지 자동 수행

## 5. 기술 스택 (Tech Stack)

| 구분 | 기술 |
|:-----|:-----|
| **데이터베이스** | MariaDB |
| **백엔드** | Django REST Framework (DRF), Python |
| **프론트엔드 (학습자)** | Vue.js 3 + Pinia + Vite 4 (Port: 5173) |
| **프론트엔드 (강사)** | Vue.js 3 + Vite 5 (Port: 5174) |
| **AI 엔진** | STT(OpenAI Whisper Chunking), LLM(GPT-4o / Claude 3.5 Sonnet) |
| **벡터 DB** | ChromaDB (RAG용) |
| **배포** | Oracle Cloud Infrastructure (OCI), Nginx, HTTPS |
| **보안** | JWT 인증(SimpleJWT), Route Guard, 404 핸들링 |

## 7. 현재 개발 완료 현황 (2026-03-13 기준)

### 구현 완료

- 사용자 인증 (JWT/SimpleJWT) 및 Role 기반 분기
- STT Chunking 수집 및 AI 요약 파이프라인 (쉬는 시간 요약)
- 일일 퀴즈 생성 및 성취도 평가
- 하이브리드 학습 모드 (오프라인, 유튜브, 유니버설 캡처)
- 학습자 대시보드 + 성취도 그래프 (Chart.js)
- Backend-as-Truth 세션 복구 시스템
- 커리어 듀얼 아웃풋 포트폴리오 (취업/창업)
- AI 모의면접 (6종 페르소나, 4차원 Rubric, 결과 리포트)
- 강사 대시보드 (학생별 퀴즈 결과 조회, 출석률 시각화)
- 분산형 수강 등록 시스템 (입장 코드)
- 실라버스 및 학습 목표 체크리스트
- STT 요약본 PDF 내보내기
- 강의 녹음 파일 업로드 → 가공 파이프라인
- AI 기본법 컴플라이언스 (사전 고지, Disclaimer, 비결정성 고지)
- OCI 배포 (HTTPS + Nginx + Django + Vue.js)

### 일부 구현 / 프로토타입

- 스킬 블록: CRUD 구현됨, 자동 생성 로직 미완성
- 동적 리라우팅: 개념 설계 완료, 알고리즘 미구현
- 갭 맵 (Gap Map): 뷰 존재, 데이터 연동 부분적
- 골든타임 알림: 이론 설계 완료, Push 인프라 미구축

### 미구현

- 게이미피케이션 (XP/뱃지/스트릭)
- 모바일 전용 앱 (PWA/네이티브)
- 멀티테넌시 (다기관 운영)
- 매니저 포탈
- 결제/구독 시스템
- 접근성 (WCAG)
