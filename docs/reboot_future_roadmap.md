# 🚀 Re:Boot 향후 로드맵 (2026 Q2 ~ 2027 Q4)

> **작성일**: 2026-03-13
> **현재 상태**: MVP v1.0 (OCI 배포 완료, 8단계 통합 테스트 통과)

---

## 🎯 핵심 요점 3가지

### 요점 1. MVP → 판매 가능한 제품 (Phase 1-2, 2026 Q2-Q3)

현재 Re:Boot은 "데모는 되지만 팔 수는 없는" MVP 단계이다. 가장 시급한 것은 **불완전한 5개 핵심 기능 완성**(PDF 포트폴리오 내보내기, 스킬 블록 자동 생성, 동적 리라우팅 v1, 골든타임 알림, PWA)과 **게이미피케이션 시스템 구축**(XP/뱃지/스트릭/리더보드)이다. 게이미피케이션은 부트캠프 수료율을 업계 평균 70-85%에서 **90% 이상으로** 끌어올리는 핵심 무기이며, 이것이 B2B 영업 시 가장 강력한 증거 자료가 된다.

### 요점 2. B2B SaaS 전환이 수익의 시작 (Phase 3, 2026 Q4)

단일 기관 도구에서 **멀티테넌시(다기관 서비스)**로 전환해야 매출이 발생한다. 멀티테넌시 아키텍처, 매니저 포탈, Stripe 결제/구독 시스템, 화이트라벨링을 구축하고, **Starter($299/월) → Growth($799/월) → Enterprise(맞춤)** 3단계 가격 체계를 도입한다. 특히 Gartner 예측 "2026년 70% B2B SaaS가 Usage-Based Pricing 채택"에 맞춰 **수료율 보장형 결과 기반 과금**(수료율 90%+ 시 할인)도 포함한다.

### 요점 3. 한국 정부 3,300만 명 AI 교육 예산이 최대 기회 (2026-2027)

한국 정부가 2030년까지 **3,300만 명** AI 교육을 목표로 천문학적 예산을 투입 중이다. Re:Boot은 **"AI로 학습을 돕는 플랫폼"**이자 **"AI 교육을 전달하는 도구"**로 이중 포지셔닝이 가능하다. 핵심 연계 포인트 3개: **(1)** 2026.06 '우리의 AI 러닝' 플랫폼에 민간 콘텐츠 공급자 등록, **(2)** 고용노동부 AI+역량 UP 프로젝트 인증, **(3)** 내일배움카드 학습자 비용 지원 연계.

## 🗺️ 6단계 로드맵 상세

### Phase 1: MVP 안정화 & 핵심 강화 (2026 Q2)

**목표**: "데모 가능한 제품"에서 "판매 가능한 제품"으로 전환

#### 1.1 PDF 포트폴리오 내보내기

- **현황**: 브라우저 `print()` 패턴으로 임시 대응
- **목표**: 서버사이드 PDF 생성 (jsPDF 또는 Puppeteer)
- **가치**: 취업 면접 제출용 실물 자산 → Re:Boot의 "자산화" 약속 이행

#### 1.2 스킬 블록 자동 생성 엔진

- **현황**: CRUD만 존재, 수동 생성 의존
- **목표**: 실라버스 섹션 + 퀴즈 결과 기반 자동 스킬 블록 부여
- **알고리즘**: `퀴즈 평균 ≥ 70% AND 출석률 ≥ 80%` → 해당 섹션 스킬 블록 자동 Earned

#### 1.3 동적 리라우팅 알고리즘 v1

- **현황**: 개념 설계 문서만 존재
- **목표**: Rule-based 리라우팅 (v1 = 규칙 기반, v2에서 ML 전환)
- **트리거 조건**: 7일 연속 미접속 / 퀴즈 3연속 50% 미만 / 세션 2주 이상 미완료
- **대응**: 커리큘럼 순서 재배열 + 압축 복구 플랜 자동 생성

#### 1.4 골든타임 알림 시스템

- **현황**: 이론 설계만 완료
- **목표**: Push 알림 (FCM) + Email (SendGrid) 이중 발송
- **핵심 로직**: `마지막 접속 후 72시간` 경과 시 학생/강사/매니저 동시 알림

#### 1.5 모바일 PWA 기초

- **현황**: 웹 반응형만 지원
- **목표**: Service Worker + Web App Manifest → 홈 화면 설치 가능
- **사유**: 네이티브 앱 대비 1/10 비용으로 모바일 접근성 확보

### Phase 2: 게이미피케이션 & 완주율 혁신 (2026 Q3)

**목표**: 부트캠프 중도 탈락률(15-30%)을 10% 이하로 낮추는 행동 설계 시스템 구축

#### 2.1 XP & 레벨 시스템

| 행동 | XP | 비고 |
|:-----|:--:|:-----|
| 세션 출석 | +50 | /일 |
| 퀴즈 100% 정답 | +200 | 난이도 가중 |
| 퀴즈 70%+ 정답 | +100 | 기본 보상 |
| AI 요약 열람 | +30 | 학습 참여 |
| 포트폴리오 생성 | +500 | 마일스톤 |
| 연속 출석 7일 보너스 | +300 | 스트릭 보너스 |
| 동료 피드백 작성 | +80 | 협업 보상 |

**레벨 체계**:
- Level 1 (0-500 XP): 신입생
- Level 2 (501-1500 XP): 탐험가
- Level 3 (1501-3500 XP): 실천가
- Level 4 (3501-7000 XP): 전문가
- Level 5 (7001+ XP): 마스터

#### 2.2 뱃지 & 마일스톤

| 뱃지 | 조건 | 유형 |
|:-----|:-----|:-----|
| 🌱 첫 발걸음 | 첫 세션 완료 | 온보딩 |
| 🔥 불꽃 학습자 | 7일 연속 출석 | 스트릭 |
| 🧠 퀴즈 마스터 | 퀴즈 10회 연속 80%+ | 성취 |
| 🛡️ 완주자 | 전체 커리큘럼 수료 | 마일스톤 |
| 💼 포트폴리오 빌더 | 첫 포트폴리오 생성 | 자산화 |
| 🚀 MVP 아키텍트 | 창업 모드 기획서 생성 | 자산화 |
| 🤝 팀 플레이어 | 동료 피드백 5회 작성 | 협업 |

#### 2.3 학습 스트릭 (연속 출석)

- Duolingo 방식 차용, Re:Boot 맥락에 맞게 변형
- 매일 최소 1개 퀴즈 완료 OR 1개 세션 시청이면 스트릭 유지
- **스트릭 프리즈** (주 1회 면제권) → 번아웃 방지
- **스트릭 복구** (72시간 내 집중 학습 시 복원) → Re:Boot 리라우팅 철학과 일치

#### 2.4 리더보드

- **개인 리더보드**: 주간 리셋, 학급 내 순위
- **팀 리더보드**: 프로젝트 팀 단위 XP 합산
- **Opt-in 참여**: 경쟁 부담 방지 (비공개 옵션 제공)

### Phase 3: B2B SaaS 전환 & 멀티테넌시 (2026 Q4)

**목표**: 단일 기관 도구에서 다(多)기관 서비스로 전환하여 B2B 매출 구조 확보

#### 3.1 멀티테넌시 아키텍처

- **Shared DB + Schema Isolation**: `django-tenants` 활용, PostgreSQL Schema 분리
- **Tenant Routing**: 서브도메인(`kitech.reboot.kr`) 또는 slug(`/org/kitech/`) 기반
- **데이터 격리**: 기관 간 학생/강사/커리큘럼 데이터 완전 분리

#### 3.2 매니저(Manager) 포탈 v1

| 기능 | 설명 |
|:-----|:-----|
| **기관 대시보드** | 전체 강좌별 수료율/이탈률/평균 성적 |
| **강사 성과 뷰** | 강사별 학생 만족도/이해도 펄스 집계 |
| **비용 분석** | AI API 사용량, 학생당 단가 |
| **보고서 내보내기** | 월간 운영 리포트 PDF/CSV |

#### 3.3 결제/구독 시스템

| 가격 모델 | 설명 | 근거 |
|:----------|:-----|:-----|
| **Tier 1 — Starter** | 월 $299 / 강사 3명, 학생 50명 | 소규모 부트캠프 |
| **Tier 2 — Growth** | 월 $799 / 강사 10명, 학생 200명 | 중견 교육 기관 |
| **Tier 3 — Enterprise** | 맞춤 견적 / 무제한 | 대규모 기관/정부 |
| **결과 기반 보너스** | 수료율 90%+ 달성 시 다음 분기 10% 할인 | Outcome-Based 트렌드 반영 |

#### 3.4 화이트라벨링

- 기관 로고/컬러 커스터마이징
- 커스텀 도메인 (`learning.kitech.re.kr`)
- 기관별 인증서/수료증 템플릿

### Phase 4: 데이터 인텔리전스 & AI 고도화 (2027 Q1)

**목표**: Rule-based AI를 ML 기반 예측 모델로 진화시켜 진정한 적응형 학습 실현

#### 4.1 리라우팅 v2 (ML 예측 모델)

| 구분 | v1 (Phase 1) | v2 (Phase 4) |
|:-----|:-------------|:-------------|
| **방식** | Rule-based | ML Gradient Boosting / LSTM |
| **입력** | 이탈 기간, 퀴즈 점수 | + 학습 패턴, 시간대, 콘텐츠 유형 선호도 |
| **출력** | 고정 복구 플랜 | 개인화 커리큘럼 + 예상 수료일 |
| **학습 데이터** | - | Phase 1-3에서 축적된 학습 로그 |

#### 4.2 AI 강의 품질 분석

- STT 음성 데이터 메타 분석 (말 속도, 반복 비율, 질문 빈도)
- **강의 난이도 자동 태깅**: 어휘 복잡도 + 학생 이해도 펄스 교차 분석
- 강사에게 "이 구간에서 이해도가 급락했습니다" 피드백 제공

#### 4.3 LLM 에이전트 튜터

| 기능 | 설명 |
|:-----|:-----|
| **24/7 질문 응답** | 당일 학습 내용(STT 로그) 기반 RAG로 즉시 응답 |
| **학습 코칭** | "오늘 퀴즈에서 이 부분을 틀렸는데, 관련 요약을 다시 볼래?" |
| **소크라테스 모드** | 답을 직접 주지 않고 힌트로 유도하는 교육적 대화 |
| **강사 부담 경감** | 반복 질문 자동 처리 → 강사는 심화 상담에 집중 |

### Phase 5: 생태계 확장 & 플랫폼화 (2027 Q2-Q3)

**목표**: Re:Boot을 단일 제품에서 교육 플랫폼 생태계로 전환

#### 5.1 커리큘럼 마켓플레이스

- 검증된 강사의 실라버스 + AI 자료를 다른 기관이 구매
- Re:Boot은 거래 수수료 15%로 플랫폼 수익 확보
- 강사의 IP(지적 재산)가 수익화되는 구조 → 강사 유입 동기

#### 5.2 개방형 API & 파트너 연동

| 연동 대상 | 방식 | 가치 |
|:----------|:-----|:-----|
| **채용 플랫폼** (원티드, 사람인) | API 포트폴리오 바로 비교 | 취업 전환율 데이터 확보 |
| **정부 HRD-Net** | 수료 데이터 자동 보고 | 국비 과정 인증 자동화 |
| **Slack/Discord** | 알림 연동 | 기존 워크플로우 유지 |
| **Zoom/Google Meet** | 화상 세션 STT 연동 | 데이터 수집 채널 확장 |

#### 5.3 커뮤니티 기능

- **코호트 내 토론방**: 같은 커리큘럼 수강생 간 질문/정보 공유
- **졸업생 네트워크**: 수료 후에도 Re:Boot 내 커리어 네트워킹
- **멘토링 매칭**: 졸업생 → 재학생 1:1 멘토링 (XP 보상)
