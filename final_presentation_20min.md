## 2. 💡 솔루션 소개 — Re:Boot란? (2분)

### 슬라이드 5: 서비스 한 줄 정의

```
Re:Boot = 학습자의 페이스메이커 + 커리어 네비게이션

"필기는 AI에게, 복습은 알고리즘에게,
 당신은 이해에만 집중하세요."
```

### 슬라이드 6~8: 3대 핵심 솔루션 (각 30초)

#### Solution 1: 몰입형 AI 학습 비서 (Focus-First)

- 실시간 STT → AI 요약 노트 자동 생성
- 문맥 인식 AI 튜터 (RAG 기반)
- "아까 선생님이 말한 그 예시 다시 설명해줘" → 수업 내용을 기억하는 AI

#### Solution 2: 데이터 기반 적응형 학습 (Dynamic Re-routing)

- 학습 부진/결석 감지 → 커리큘럼 자동 재설계
- 취약 구간 AI 보충 자료 자동 생성
- 에빙하우스 5주기 간격 반복 복습

#### Solution 3: 학습 경험의 자산화 (Assetization)

- 중도 포기 ≠ 실패 → **부분적 성취(스킬블록)**로 자산화
- 포트폴리오 자동 생성 + 갭 맵 비교

## 3. 🏗️ 시스템 아키텍처 & 기술 스택 (2분)

### 슬라이드 9: 기술 스택 요약 테이블

|     영역      | 기술                         | 핵심 역할                                           |
| :-----------: | ---------------------------- | --------------------------------------------------- |
| **Frontend**  | Vue.js 3 + Vite + Pinia      | Glassmorphism UI, 2개의 독립 SPA (학생/교수)        |
|  **Backend**  | Django REST Framework        | 50+ REST API, JWT 인증, 역할 기반 접근 제어         |
| **Database**  | PostgreSQL + pgvector        | 벡터 유사도 검색으로 RAG 구현                       |
| **AI Engine** | GPT-4o / GPT-4o-mini         | 요약·퀴즈·튜터링·형성평가·복습루트 등 9가지 AI 기능 |
|    **STT**    | Web Speech API + Whisper     | 실시간 + 녹음 파일 듀얼 파이프라인                  |
|   **Infra**   | OCI + Nginx + Gunicorn + SSL | HTTPS 배포, 프로덕션 운영                           |

> **대본 포인트**: "학생 앱과 교수 대시보드를 **완전히 분리된 2개의 독립 Vue 앱**으로 구축했습니다."

### 슬라이드 10: 시스템 아키텍처 다이어그램

```
[학생 앱 :5173] ──┐
                  ├── [Nginx/SSL] ── [Gunicorn] ── [Django REST API] ── [PostgreSQL + pgvector]
[교수 앱 :5174] ──┘                                       │
                                                    [OpenAI API]
                                                    GPT-4o / Whisper / Embedding
```

> **강조점**: 프론트-백-AI-DB 4계층 분리 아키텍처

### 슬라이드 11: 학습 여정 흐름도

```
수준 진단 → Level 분류 → 적응형 교안 → 라이브 수업(STT+펄스)
  → AI 퀴즈 → 통합 노트 → 형성평가 → 복습 루트 → 간격 반복 → 스킬블록 획득
```

> **대본**: "학생이 처음 들어와서 수준 진단을 받고, 수업을 듣고, 복습하고, 스킬을 축적하기까지의 **전체 학습 라이프사이클**을 하나의 플랫폼 안에서 관리합니다."

## 4. 🖥️ 라이브 데모 — 학생 여정 (5분) ⭐ 핵심 파트

> [!IMPORTANT]
> 이 파트가 발표의 **하이라이트**입니다. 실제 화면을 보여줌으로써 "말만 하는 기획"이 아닌 "작동하는 서비스"를 증명합니다.
> 시연이 불가능한 환경이라면, 미리 녹화한 영상 또는 **스크린샷 캐러셀**로 대체하세요.

### Scene 1: 로그인 & 대시보드 (30초)

**시연 순서**:

1. 학생 계정(`student1`)으로 로그인
2. 메인 대시보드 진입 — 학습 시간, 수강 수업 수, 평균 퀴즈 점수, 출석률이 한눈에 표시
3. Glassmorphism UI의 세련된 디자인 강조

> **대본**: "로그인하면 바로 이 대시보드가 뜹니다. 총 학습 시간, 퀴즈 점수, 출석률 — 내 학습 현황이 한눈에 보입니다."

### Scene 2: 수준 진단 (Placement Test) (30초)

**시연 순서**:

1. 수준 진단 테스트 진입
2. 3개 카테고리(프로그래밍 기초, 데이터 구조, 웹 기초) × 3개 난이도의 문제 풀이
3. 결과: Level 1(초급) / Level 2(중급) / Level 3(고급) 자동 분류
4. 갭 맵 시각화 (OWNED / LEARNING / GAP)

> **대본**: "처음 들어오면 수준 진단을 받습니다. 결과에 따라 Level이 분류되고, 어떤 스킬이 부족한지 '갭 맵'으로 시각화됩니다."

### Scene 3: 라이브 수업 참여 (1분 30초) ⭐ 가장 중요

**시연 순서**:

1. 입장 코드 입력 → 라이브 세션 입장
2. **실시간 STT** — 마이크 녹음 시작, 화면에 텍스트가 실시간으로 올라오는 모습
3. **이해도 펄스** — ✅(이해했어요) / ❓(잘 모르겠어요) 버튼 클릭
4. **AI 체크포인트 퀴즈** — 수업 중 자동 생성된 4지선다 퀴즈에 응답
5. **RAG 챗봇** — "아까 배운 변수와 자료형의 차이가 뭐야?" 질문 → 수업 내용 기반 답변

> **대본**: "지금 보시는 것처럼 강의가 진행되면 실시간으로 텍스트가 변환됩니다. 필기할 필요가 없습니다. 이해가 안 되는 부분은 이해도 버튼으로 표시하고, AI 챗봇에게 바로 질문할 수 있습니다. 이 챗봇은 일반 ChatGPT와 다르게, **오늘 수업 내용만** 참조하여 답변합니다."

### Scene 4: 세션 종료 → AI 통합 노트 (1분)

**시연 순서**:

1. 세션 종료 버튼 클릭
2. **GPT-4o가 자동 생성한 통합 노트** 확인
   - 수업 개요, 핵심 내용 정리, 퀴즈 복습, Q&A, 핵심 키워드, 복습 포인트
3. 노트 내에서 메모 추가 (Append) 기능
4. PDF 내보내기 버튼 시연

> **대본**: "수업이 끝나면 GPT-4o가 STT 전문, 퀴즈 결과, 여러분의 질문, 이해도 데이터까지— **7가지 데이터를 통합**하여 이런 깔끔한 강의 노트를 자동으로 만들어줍니다."

### Scene 5: 복습 & 간격 반복 (30초)

**시연 순서**:

1. 복습 루트 확인 — AI가 제안한 개인화된 복습 경로
2. 간격 반복 — 에빙하우스 5주기(10분→1일→1주→1개월→6개월) 스케줄

> **대본**: "틀린 문제와 이해도 데이터를 기반으로 AI가 '오늘 복습해야 할 항목'을 알려줍니다. 에빙하우스 망각곡선에 따라 최적의 타이밍에 반복합니다."

### Scene 6: 스킬블록 & 갭 맵 (30초)

**시연 순서**:

1. 스킬블록 카드 목록 (🌱씨앗 → 🌿새싹 → 🌸꽃 레벨별 이모지)
2. 갭 맵 vs 현재 보유 스킬블록 비교 화면

> **대본**: "수업을 듣고 퀴즈를 통과할 때마다 '스킬블록'이 쌓입니다. 설령 과정을 끝까지 수료하지 못하더라도, 이 스킬블록이 여러분의 **학습 자산**으로 남습니다."

## 5. 📊 교수자 대시보드 시연 (3분)

### 슬라이드 12: 교수자 대시보드 소개 (20초)

> "Re:Boot의 또 다른 핵심은 **교수자용 대시보드**입니다. 학생 개개인의 학습 상태를 데이터로 파악하고, AI의 도움을 받아 능동적으로 개입할 수 있습니다."

### Scene 7: 강의 관리 & 모니터링 (40초)

**시연 순서**:

1. 교수 계정(`professor`)으로 로그인
2. 강의 목록 → 강의 상세 진입
3. **7개 탭** 간략 소개: 진도 모니터링, 출석, 퀴즈 분석, 학습 분석 등

### Scene 8: 학습 분석 대시보드 (1분 30초) ⭐

**시연 순서**:

1. **현황판**: 레벨 분포 도넛 차트 + 위험군 학생 자동 감지 테이블
2. **취약 구간 인사이트**: 퀴즈+형성평가 통합 오답률 분석, 차시별 비교 차트
3. **AI 제안 승인**: WeakZone 보충자료, 적응형 교안, 복습루트 등 AI 제안을 교수가 검토 후 승인/거부
4. **품질 리포트**: 7가지 강의 품질 메트릭, 레벨 재분류 제안

> **대본**: "위험군으로 감지된 학생을 클릭하면, AI가 자동으로 생성한 보충 자료나 맞춤 교안을 교수자가 검토하고 승인할 수 있습니다. AI가 독단적으로 결정하는 것이 아니라, **교수자가 최종 결정권**을 갖습니다."

### Scene 9: 그룹 메시지 발송 (30초)

**시연 순서**:

1. 레벨별 또는 개별 학생 선택
2. 메시지 작성 → 발송
3. (학생 앱으로 전환) 메시지 수신 + 읽음 처리 확인

> **대본**: "교수자는 위험군 학생들에게 격려 메시지를 보내거나, 레벨별로 맞춤 안내를 할 수 있습니다."

### 슬라이드 17: 확장 비전

> - **학습자 (B2C)**: '막막함'이라는 심리적 장벽 제거 + 실패 없는 커리어 전환
> - **사회적 가치**: 학습 실패를 데이터 자산으로 전환 → 언제든 다시 도전할 수 있는 건강한 교육 생태계
