💡 시스템 구조 요약

Re:Boot 시스템은 크게 3가지 계층(Layer)으로 나뉘어져 있습니다.

  • 프론트엔드 (사용자 화면): 학생과 교수님이 직접 보고 클릭하는 화면입니다. (학생용 웹앱, 교수자용 대시보드)
  • 백엔드 (서버 & 비즈니스 로직): 사용자의 요청을 받아 다양한 기록을 관리하고 AI와 소통하는 두뇌 역할입니다. (Django 서버)
  • 인프라 & 외부 연동 (외부 시스템): 데이터를 저장하는 창고(DB)와 똑똑한 AI 기능을 제공하는 외부 서비스(OpenAI API 등)입니다.

🔑 각 연결 흐름의 주요 역할

  1. 학생/교수님 ↔ 백엔드: 사용자가 브라우저에서 버튼을 클릭하거나 페이지를 이동하면 모든 요청이 서버의 '요청 분배기(API)'로 전달됩니다.
  2. 백엔드 ↔ 데이터베이스 (PostgreSQL): 진도율, 로그인 정보, 설정 등 사라지면 안 되는 핵심 정보들은 모두 데이터베이스(창고)에 영구적으로 안전하게 보관됩니다.
  3. 백엔드 ↔ AI 엔진 (OpenAI): "학습 복구 플랜", "퀴즈 자동 생성", "AI 요약본" 같은 고도화된 기능은 백엔드가 서버에 연동된 똑똑한 GPT-4o 모델에 질문을 던져 답변을 받아 화면에 뿌려줍니다.
  4. 백엔드 ↔ STT 엔진: 교실에 있는 음성 인식 장치(라스베리파이 등)가 실시간으로 교수님의 목소리를 글자(자막)로 변환해 라이브 세션 모듈로 전송합니다.
