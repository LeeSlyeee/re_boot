# Re:Boot 서비스 기능 설계서 (Full Detailed Specification)

## 0. 서비스 개요 및 철학
- **서비스 명칭**: Re:Boot
- **작성 목적**: 팀원 간 기능 구현 우선순위 공유 및 학습자용·교수자용 페이지 설계 기준 정리
- **서비스 성격**: 현장 대면 강의 기반 실시간 학습 지원 플랫폼 (녹화 강의 아님)
- **사용자**: 학습자(Learner) / 교수자(Instructor) --- 두 개의 독립된 페이지로 구성
- **슬로건**: "학습자는 자신이 어디에 있는지 알고, 교수자는 어디로 보낼지 결정한다."
- **핵심 목표**: 부트캠프 현장 대면 강의를 디지털화하여, 학습자가 자신의 현재 수준과 목표 사이의 갭을 명확히 인식하고 그 갭을 메울 수 있도록 지원. 교수자는 AI 데이터를 기반으로 최종 판단을 내리는 커리큘럼 디렉터 역할을 수행.
- **설계 원칙**:
  1. 기능을 추가하지 않는다. 핵심을 고도화한다.
  2. 데이터 파이프라인 순서를 지킨다: **진단 → 세션 → 강의 → 대시보드**
  3. 현장 강의의 흐름을 방해하지 않는다. UI는 탭 1~2번으로 조작 완료되어야 한다.
  4. 교수자의 전문적 판단이 항상 AI 제안보다 우선한다.

## 1. 전체 Phase 구조
- **Phase 0. 라이브 세션 인프라**: 현장과 디지털을 연결하는 뼈대 (모든 기능의 전제 조건)
- **Phase 1. 수준 진단 + 갭 맵 시각화**: 학습자가 자신의 위치를 인식
- **Phase 2. 강의 기능 고도화**: 수준에 맞는 학습 경험 제공
- **Phase 3. 교수자 대시보드**: 데이터 기반 권한자 역할 완성

⚠️ **주의**: Phase 0이 없으면 Phase 1~3은 작동하지 않습니다.

## 2. Phase 상세 설계

### Phase 0. 라이브 세션 인프라 (현장 디지털화)

#### 0-1. 세션 생성 + 입장
- **교수자 페이지**:
  - 수업 시작 전 "오늘 수업 시작" 버튼 탭 (강의명/날짜/차시 자동 기록)
  - 6자리 세션 코드 및 QR코드 자동 생성 → 학습자에게 공유
  - 접속 중인 학습자 목록 실시간 확인
- **학습자 페이지**:
  - QR 스캔 또는 6자리 코드 입력으로 입장
  - 본인의 현재 레벨(Phase 1 결과) 세션 자동 연동
  - 입장 화면: 오늘 강의 주제, 나의 현재 레벨 표시

#### 0-2. 실시간 이해도 펄스 체크 (Pulse Check)
- **학습자 페이지**: 하단 항상 고정 (✅ 이해했어요 / ❓ 잘 모르겠어요), 익명 전송, 변경 가능
- **교수자 페이지**: 실시간 이해도 비율 바(Bar) 표시. 50% 미만 시 빨간색 경고 알림. 전체 반응 한눈에 파악. 세션 종료 후 취약 구간 데이터로 저장.

#### 0-3. 교수자 주도 체크포인트 발동 (Live Checkpoint Trigger)
- **STT 기반 즉석 퀴즈 프로세스**:
  1. **실시간 텍스트 스트리밍 (Short-term STT)**: 30초~1분 단위 음성 발췌 및 텍스트 변환
  2. **AI 퀴즈 후보 자동 생성 (On-the-fly Generation)**: 핵심 용어(예: 클로저, 스코프) 포착 및 퀴즈/정답/해설 생성
  3. **최종 검토 및 발동**: 교수자 스마트 팝업 확인 → "확인" 탭 1번으로 전체 전송
- **🛠️ 기술적 보완책**: 도메인 사전 학습(JS, Python 등), 난이도 자동 조절(Phase 1 데이터 기반)

#### 0-3 부록. STT 3단계 아키텍처
1. **키워드 스팟팅 (Keyword Spotting)**: "이해하셨나요?", "잠깐 퀴즈!" 등 트리거 감지. 저부하, 지연시간 거의 없음.
2. **배치 전사 (Batch Transcription)**: 수업 종료 후 일괄 처리(Whisper 등). 정확도 높음, IT 전문 용어 교정.
3. **핵심 역할**: 강의 흐름을 끊지 않는 "탭 1번" 비서 역할.

#### 🛠️ STT 구현 필수 체크리스트
- 음성 수집 사전 동의 확보 (법적 요건)
- 전용 라발리에 마이크 사용 권장
- Re:Boot 전용 기술 용어 사전 구축 및 AI 학습

#### 0-4. 익명 질문 큐 (Anonymous Q&A Queue)
- **학습자**: 텍스트 질문 작성, 완전 익명 처리, 답변 상태 확인
- **교수자**: 질문 큐 표시, 유사 질문 자동 묶음 처리, 답변 또는 예약(쉬는 시간)

#### 0-5. 세션 종료 + 자동 요약 생성
- **교수자**: "수업 종료" 탭 → 세션 마감 + 배치 전사 실행 → 인사이트 리포트 생성 (최저 이해 구간, TOP 질문 등)
- **학습자**: 개인 요약 표시 (퀴즈 통과율, 갭 맵 업데이트 알림, 어려웠던 개념 및 추천 보충 자료)

#### 0-6. AI 강의 요약 노트 자동 생성 + 교안 통합 (Secondary Knowledge Artifact)
- **⚙️ 가공 파이프라인**: 배치 전사 → 핵심 개념 추출(LLM) → 교안 매핑 → 교수자 검토 → 학습자 배포
- **교수자 설정**: 강의 전 교안(PDF, PPT, MD 등) 업로드 → 학습 목표 등록
- **통합 요약 노트 구성**: 교안 원본 + 라이브 추가분(🎙️ 표시, 예시/비유/코드) + 핵심 키워드
- **🔒 지적재산권 보호**: 원본 음성 자동 파기(기본 24시간), 2차 가공물만 배포, 교수자 검토/비공개 권한.
- **📌 결석생 시나리오**: 요약 노트 발송 → 셀프 테스트 → 갭 맵 반영 → 교수자 확인 가능

### Phase 1. 수준 진단 + 갭 맵 시각화

#### 1-1. 입학 진단 테스트 (Placement Quiz)
- 최초 가입 직후 및 주기적 재진단. 개념/실습/학습 패턴 15~20문항.
- **결과**: Level 1(초보), Level 2(기초), Level 3(심화) 분류.

#### 1-2. 목표 설정 (Goal Anchoring)
- 직무/직종 선택 (역량 정보 제공) → 리스트 형태 선택 → 예상 도달 기간 및 필요 역량 리스트 미리보기.

#### 1-3. 갭 맵 시각화 (Gap Map)
- 역량 블록 시각화 (✅ 보유 / 🔲 미보유 / 🔄 학습 중).
- 상세 정량적 수치(프로그래스 바) 표현. 체크포인트 데이터 기반 자동 업데이트.

---

### Phase 2. 강의 기능 고도화

#### 2-1. 부족 구간 알림 (Weak Zone Alert)
- 오답/불이해 연속 발생 시 알림 및 보충 자료 제공. 교수자 승인 기반 푸시.

#### 2-2. 수준별 콘텐츠 분기 (Adaptive Content Branching)
- 교안을 학습자 레벨(1/2/3)에 맞춰 변형 제안 → 교수자 선택/수정.
- 학습자는 본인 레벨 자료 자동 표시 + 상위 레벨 도전 옵션.

#### 2-3. AI 복습 루트 제안 (AI Review Suggestion)
- **5주기 암기법(에빙하우스)** 적용: 10분 후 / 1일 후 / 1주일 후 / 1개월 후 / 6개월 후.
- 오답 개념 → 이전 강의 → 내일 강의 순서 제안. 통합 요약 노트 우선 연결.

#### 2-4. 요약 노트 기반 형성평가 연계
- 요약 노트 배포 시 사후 퀴즈(3~5문항) 생성 및 승인.
- **📌 간격 반복(Spaced Repetition)**: 오답 개념을 1일, 3일, 7일 후 미니 퀴즈(인출 연습) 알림 발송.

---

### Phase 3. 교수자 대시보드 (데이터 기반 권한자 역할)

#### 3-1. 학습자 수준 현황판
- 레벨 분포, 진도율, 출석률 시각화. 위험군(결석/오답률/불이해) 표시. 결석생 보충 학습 확인.

#### 3-2. 취약 구간 인사이트
- 누적 데이터 통합 분석. 오답 반복 구간 및 강의 개선 포인트 우선순위 정렬.

#### 3-3. AI 제안 승인 흐름
- AI의 보충 자료/복습 루트 제안에 대해 교수자 승인/교체/거부.

#### 3-4. 그룹별 개입 도구 + 강의 품질 리포트
- 레벨별/상태별 그룹 재분류 및 타겟 피드백. 차시별 비교 추이 리포트.

## 3. 스킬블록 연결 (Phase 3 이후)
- 체크포인트 + 이해도 + 형성평가 기반 공용 스킬블록 자동 생성.
- Gamification: 레벨별 이모지(🌱 씨앗 / 🌿 새싹 / 🌸 꽃). 모의면접 정량 데이터 연계.

---

## 4. 기술 구현 고려 사항
- 스마트폰/노트북 (모바일/웹) 동시 지원.
- 오프라인 응답 후 재연결 시 자동 동기화.
- UI 조작 최소화 (1~2회 탭).
- 개인정보(익명 처리 IRB 요건) 및 음성 데이터 보안(암호화, 파기 정책).

### Phase 1 — 수준 진단 + 갭 맵

- [x] [NEW] 교수자: 요약 노트 기반 사후 형성평가 문항 생성 + 승인
