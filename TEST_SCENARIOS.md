```bash

cd backend && source ../venv/bin/activate && python manage.py runserver

# 터미널 2: 학생 프론트엔드
cd frontend && npm run dev          # → http://localhost:5173

# 터미널 3: 교수자 대시보드
cd Professor_dashboard && npm run dev  # → http://localhost:5174
```

## 📋 시나리오 1: 라이브 세션 풀 플로우 (Phase 0 핵심)

> 예상 소요: 10~15분
> 검증 대상: 세션 생성 → 입장 → 펄스 → 퀴즈 → Q&A → 종료 → 노트

### 교수자 (localhost:5174)

1. **로그인** → `professor` / `password123`
2. **강의 선택** → 강의 상세 페이지 진입
3. **라이브 탭** 클릭
4. **세션 생성** → 제목 입력 (예: "테스트 수업") → [세션 생성] 클릭
5. **확인 항목**:
   - [ ] 6자리 입장 코드 표시
   - [ ] QR 코드 이미지 표시
   - [ ] `📅 N차시` 배지 표시 (**A2**)
   - [ ] 상태 "⏳ 대기 중" 표시

6. **세션 시작** → [세션 시작] 버튼 클릭

### 학생 (localhost:5173)

7. **로그인** → `testuser` / `password123`
8. **학습 시작** → [라이브 세션 참여] 클릭
9. **코드 입력** → 교수자 화면에 표시된 6자리 코드 입력 → [입장하기]
10. **확인 항목**:
    - [ ] "🟢 테스트 수업" 제목 + "진행 중" 배지 표시
    - [ ] 세션 코드 표시
    - [ ] 📷 QR 코드 스캔 버튼 존재 (**C3**)

### 교수자 확인

11. **참가자 목록 확인**:
    - [ ] `testuser` 표시
    - [ ] 레벨 배지 `Lv1/2/3` 표시 (**A1**) — 진단 데이터 있으면

18. **확인 항목**:

### 학생

## 📋 시나리오 3: 실시간 Q&A + 클러스터링 (Phase 0 + B1)

### 학생

22. **Q&A 패널 열기** → 하단 [💬 Q&A] 버튼 클릭
23. **질문 입력** → "변수의 스코프가 뭔가요?" → [전송]

    - [ ] 질문 2개 표시됨
    - [ ] (백그라운드) 유사 질문 클러스터링 실행됨 (**B1**)

26. **Q&A 확인** → 실시간 질문 패널에 학생 질문 표시

    - [ ] 학생 화면에 교수자 답변이 "👨‍🏫 교수자" 배지와 함께 표시

## 📋 시나리오 4: 세션 종료 + 노트 생성 (Phase 0 + B2/B3/C2)

### 교수자

29. **세션 종료** → [세션 종료] 클릭
30. **대기** → AI 노트 + 인사이트 자동 생성 (약 10~30초)
31. **확인 항목**:
    - [ ] 인사이트 리포트 표시
    - [ ] 통합 노트 내용에 🎙️ 교수자 설명 라벨 포함 (**B3**)
    - [ ] 교안 연결 교안 매핑 반영 (**C2**) — 교안 업로드 된 경우

### 노트 승인

32. **교안 연결** → 체크박스로 교안 선택 → [📎 교안 연결 저장]
33. **배포 범위 선택** → 드롭다운에서 범위 선택 (**B4**)
    - [ ] 🌐 전체 공개 / ✅ 출석자만 / 📩 결석생만 / Level 1~3 옵션 존재
34. **결석생 공개 체크** → ☑️ 체크
35. **[✅ 노트 승인 → 학생 공개]** 클릭
36. **확인 항목**:
    - [ ] "✅ 승인 완료" 배지 표시
    - [ ] alert에 배포 범위 표시 (예: "범위: ALL")

### 학생

37. **노트 확인** → 세션 종료 화면에서:
    - [ ] 📊 내 수업 요약 카드 표시 (**B2**)
    - [ ] 퀴즈 정답률, 혼란 비율 표시
    - [ ] 오답 개념 칩 표시 (있을 경우)
    - [ ] 📚 통합 노트 내용 표시

## ⚡ 빠른 API 테스트 (curl)

서버가 돌고 있을 때, 아래 명령어로 개별 API를 빠르게 확인할 수 있습니다.

```bash
# 1. 토큰 발급
TOKEN=$(curl -s http://localhost:8000/api/accounts/token/ \
  -H "Content-Type: application/json" \
  -d '{"username":"professor","password":"password123"}' | python3 -c "import sys,json; print(json.load(sys.stdin)['access'])")

echo "Token: $TOKEN"

# 2. 라이브 세션 목록
curl -s http://localhost:8000/api/learning/live/ \
  -H "Authorization: Bearer $TOKEN" | python3 -m json.tool

# 3. 분석 대시보드 (lecture_id=1 기준)
curl -s http://localhost:8000/api/learning/professor/1/analytics/overview/ \
  -H "Authorization: Bearer $TOKEN" | python3 -m json.tool

# 4. 취약구간 인사이트
curl -s http://localhost:8000/api/learning/professor/1/analytics/weak-insights/ \
  -H "Authorization: Bearer $TOKEN" | python3 -m json.tool

# 5. AI 제안 목록
curl -s http://localhost:8000/api/learning/professor/1/analytics/ai-suggestions/ \
  -H "Authorization: Bearer $TOKEN" | python3 -m json.tool

# 6. 품질 리포트
curl -s http://localhost:8000/api/learning/professor/1/analytics/quality-report/ \
  -H "Authorization: Bearer $TOKEN" | python3 -m json.tool

# 7. 학생 토큰으로 개인 요약 (session_id 필요)
STOKEN=$(curl -s http://localhost:8000/api/accounts/token/ \
  -H "Content-Type: application/json" \
  -d '{"username":"testuser","password":"password123"}' | python3 -c "import sys,json; print(json.load(sys.stdin)['access'])")

curl -s http://localhost:8000/api/learning/live/{SESSION_ID}/my-summary/ \
  -H "Authorization: Bearer $STOKEN" | python3 -m json.tool

# 8. 결석생 셀프 테스트 (note_id 필요)
curl -s -X POST http://localhost:8000/api/learning/absent-notes/{NOTE_ID}/self-test/ \
  -H "Authorization: Bearer $STOKEN" | python3 -m json.tool
```
